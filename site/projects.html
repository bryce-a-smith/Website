Monitoring & Alerting Pipeline

Multi-environment monitoring and alerting pipeline. UptimeRobot + Power Automate + GitHub Actions + AWS Amplify. Live status page. Includes a real incident postmortem from a production outage during setup. (Still a work in progress)
Stack: GitHub Actions · AWS Amplify · Route 53 · UptimeRobot · Power Automate · JavaScript

Read more


Designed and deployed a lightweight observability stack for my personal infrastructure — because if you're going to talk about production reliability, your own site should have monitoring. The pipeline covers three environments (dev, qa, prod) with automated health checks every five minutes, webhook-driven alerting on status changes, and a live status page that shows deployment state and last commit timestamp in real time.
Built on GitHub Actions, AWS Amplify, UptimeRobot, and Power Automate.

The more interesting part: setting it up caused a two-hour outage on aldenbryce.com — a DNS conflict between two AWS Amplify apps claiming the same CloudFront distribution, compounded by a missing IAM service role. I documented the full incident response from detection through root cause analysis to resolution. That writeup lives at /incidents and is probably a more useful portfolio artifact than the project itself.

Incident Postmortems

Built an incident management workflow for my personal infrastructure — from detection through documentation. Incidents are stored in a version-controlled JSON file and rendered client-side with no backend required. Each entry includes a structured root cause analysis and remediation timeline, modeled after standard SRE postmortem format. The log is publicly visible on the site and doubles as a portfolio artifact demonstrating operational thinking alongside the technical work.
Stack: HTML · CSS · JavaScript · AWS Amplify · GitHub Actions
